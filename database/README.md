# Database Setup

This folder contains the database schema and sample data for the Study Planner application.

## Problem Context

The Study Planner needs persistent data storage for all assignments. This database folder provides the schema definition and initial data needed to set up SQLite. The generated SQL files allow reproducible database initialization and can be imported into other database systems for deployment.

## Files

- **schema.sql** - Database table definitions for SQLite (auto-generated export)
- **seed.sql** - Sample assignments for testing (optional)

## Automatic Setup (Recommended)

The database is **automatically created** when you run the backend for the first time. No manual setup needed:

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload

# ✓ database.db is created automatically
# ✓ Tables are created automatically
# ✓ Ready to use!
```

The backend uses SQLAlchemy ORM which automatically creates the database schema defined in `backend/models.py`.

## Manual Setup (Optional)

If you prefer to initialize the database manually using SQL files:

```bash
cd backend

# Create database from schema.sql
sqlite3 database.db < ../database/schema.sql

# (Optional) Load sample data
sqlite3 database.db < ../database/seed.sql

# Then run backend
uvicorn main:app --reload
```

## Reset Database

To completely reset and start fresh:

```bash
cd backend
rm database.db                  # Delete existing database
uvicorn main:app --reload      # Creates fresh database automatically
```

To reload sample data after reset:

```bash
sqlite3 database.db < ../database/seed.sql
```

## Database Schema

```
assignments
├── id (INTEGER) - Primary Key
├── title (VARCHAR) - Assignment title
├── description (TEXT) - Assignment details/notes
├── subject (VARCHAR) - Subject/course name
├── priority (VARCHAR) - Low / Medium / High
├── status (VARCHAR) - Pending / Completed
├── due_date (DATE) - Assignment due date
├── created_at (DATETIME) - When assignment was created
└── updated_at (DATETIME) - When assignment was last updated
```

## Notes

- Currently uses **SQLite** (file-based database)
- For production deployment, migrate to **PostgreSQL**
- All ID values are auto-generated by SQLAlchemy

## Challenges Overcome

**SQLite Setup in VS Code Environment:** Setting up SQLite in VS Code required configuring Python virtual environments, managing SQLite binary paths, and ensuring database files had proper read/write permissions. The solution implemented automatic database initialization through SQLAlchemy ORM, eliminating the need for users to manually execute SQL scripts.

**Database Export & Version Control:** Exporting schema.sql and seed.sql files for submission required understanding SQLite dump formats and ensuring these files could be re-imported without errors. This allows graders to initialize a fresh database from the exported schema and test the application with sample data.

**Cross-Platform Compatibility:** Database files and setup scripts needed to work across Windows, macOS, and Linux systems with different path formats and shell commands. The README provides platform-specific instructions for each operating system.
